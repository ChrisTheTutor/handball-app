01 — Why This Exists

The Vision

Handball has no real digital home. Players find games through word of mouth, group chats, and showing up and hoping someone's there. This app fixes that. It starts with NYC — the handball capital — and gives players a real platform to organize, compete, and document their game.

The Problem

What exists today
  • Games organized through group chats and word of mouth
  • No way to find open games near you
  • Scores forgotten, never tracked
  • No tournament infrastructure for casual players
  • No way to attach video to a game
  • No player identity or stats over time

The Solution

What this app does
  • Map of open games near you, right now
  • Create a game in 30 seconds, share a link
  • Official game sheet filled out by a referee
  • Tournament brackets built and managed in-app
  • Attach YouTube or video link to any game
  • Player profile with full game and score history

The Edge

Why this wins
  • Built by a handball player, for handball players
  • Referee game sheet is unique — no other app has this
  • Video attachment turns casual games into content
  • NYC community is tight — word spreads fast
  • Foundation ready to expand to all sports
  • You know the courts, the players, the culture
🎯 The one sentence pitch: "Find a handball game near you in NYC, log your score, run a tournament, and upload the highlight reel — all in one place."
02 — What It Does

Core Features

Six core features for the MVP. Each one is independently valuable — players will use the app even if they only care about one of them.

Find Open Games

Discovery · Map-based

A map of NYC showing open games happening today and this week. Filter by court, game type (singles, doubles, cutthroat), skill level, and time. Tap a game to see who's playing and join with one tap.

  • Map view with court pins showing active games
  • List view — sorted by distance or time
  • Filter by: game type, skill level, time, court
  • Join with one tap — notifies the game creator
  • See player profiles of who's already in

Create a Game

Organizer · 30-second setup

Create an open game in under 30 seconds. Set the court, time, game type, and how many spots are open. Share a link directly to teammates via text or social. The game appears on the map instantly.

  • Pick court from saved NYC locations
  • Set date, time, game type, max players
  • Set skill level (open, intermediate, advanced)
  • Shareable link for non-app users to view
  • Creator gets notified when players join

Score Logging + Referee Sheet

Records · Official Game Sheet

After any game, log the score. For official games, assign a referee who fills out a full game sheet — points per rally, serving rotation, timeouts, player fouls. Like a baseball scorecard but for handball. This data feeds each player's career stats.

  • Quick score log — winner, score, players
  • Full referee mode — point-by-point entry
  • Serving rotation tracker
  • Player fouls and timeouts logged
  • Game sheet saved permanently to both profiles
  • Stats auto-update on player profiles

Video Attachment

Highlights · YouTube + Links

Attach a YouTube link, Instagram Reel, or any video URL to a completed game. The video shows up on the game page, on both player profiles, and in a public highlights feed. Turns every filmed game into a permanent record.

  • Attach YouTube, Instagram, or any video URL
  • Video preview thumbnail generated automatically
  • Shows on game page and both player profiles
  • Public highlights feed — best games in NYC
  • Multiple videos per game allowed

Tournaments

Competition · Brackets + Sign-ups

Create a tournament in minutes. Set the format (single elimination, round robin), court locations, date, and entry requirements. Players sign up in-app. Brackets auto-generate and update in real time as scores are logged.

  • Create: name, format, court, date, max players
  • Public sign-up page shareable by link
  • Auto-generated bracket on sign-up close
  • Score entry updates bracket in real time
  • Referee sheet available for each match
  • Tournament history saved on all player profiles

Player Profiles

Identity · Stats + History

Every player gets a profile showing their full game history, win/loss record, tournament results, and highlight videos. Your handball identity — built automatically as you play. Searchable by other players.

  • Win/loss record — overall and by game type
  • Tournament history and placements
  • Recent games with scores and opponents
  • Highlight video reel
  • Home courts and preferred game types
  • Follow other players — see their activity

05 — Technology

Tech Stack

Chosen for speed of development, not complexity. Every tool here has great documentation and a large community — meaning when you get stuck, answers are easy to find.

Frontend
React Native + Expo
One codebase runs on Android phone and browser. Expo removes the complex native setup. You write JavaScript — approachable with some coding experience.
Backend / Database
Supabase
Postgres database + authentication + real-time updates + file storage — all in one. Free tier is generous. No server to manage. SQL is readable and learnable. Powers both apps.
Authentication
Supabase Auth
Email + Google login built in. One line of code. Handles security so you don't have to think about it.
Maps
Google Maps API
Best map quality for NYC. React Native has a clean Google Maps wrapper. You pre-load all NYC handball courts as data — no map scraping needed.
Video Embeds
YouTube oEmbed API
Free API. Paste any YouTube URL and get a thumbnail + embed automatically. Works for Instagram and other platforms via open graph data.
Notifications
Expo Push Notifications
Built into Expo. Send push notifications when players join your game, tournament brackets update, or scores are submitted. Free.
AI Assist
Claude API
For the math homework tool and eventually smart scheduling in your Life OS. You already know how to use Claude — plug the API in when ready.
Deployment
Expo + Vercel
Expo publishes the mobile app. Vercel hosts the web version. Both deploy with one command. Both free to start.
💡 Why not Flutter, Swift, or Kotlin? React Native means one codebase for Android + web. Since you have some coding experience and will be using AI to help build, JavaScript is the most accessible path. You can always add native modules later if needed.
06 — Execution

Build Plan

Three phases. Phase 1 gets you something real people can use. Phase 2 adds the features that make it sticky. Phase 3 opens it up to grow.

Phase 1 — Weeks 1–3 · Foundation
Get It Working
Auth, profiles, create game, find game on map, join game. By end of week 3 you should be able to create a real game and text a friend the link to join.
  • Set up Supabase project — database, auth, tables
  • Set up Expo React Native project
  • User registration and login (email + Google)
  • Basic player profile screen
  • Pre-load all NYC handball courts into database
  • Create game form — court, date, time, type, spots
  • Map screen showing open games as pins
  • Join game with one tap
  • Push notification when someone joins your game
Phase 2 — Weeks 4–6 · The Features That Matter
Score, Ref, and Film
This is where your app becomes unique. Quick score logging, the referee game sheet, and video attachment. By end of week 6 you can play a game, have someone ref it, log the score, and attach the YouTube clip — all in one place.
  • Quick score log — winner, score, confirm both players
  • Referee game sheet — point-by-point entry UI
  • Serving rotation tracker in ref mode
  • Fouls and timeouts log
  • Game sheet saves to both player profiles
  • Video URL attachment to any game
  • YouTube thumbnail auto-generation
  • Highlights feed — all NYC game videos
  • Player profile — stats auto-update from game data
Phase 3 — Weeks 7–8 · Tournaments + Polish
Run the First Tournament
Tournament creation, sign-up, bracket generation, and real-time updates. Then polish, bug fix, and invite your handball community to use it. The goal: run one real NYC tournament through the app before end of summer.
  • Tournament creation — name, format, court, date, max players
  • Public sign-up page with shareable link
  • Auto-bracket generation on sign-up close
  • Bracket UI — visual bracket that updates live
  • Match scheduling within tournament
  • Tournament results on all player profiles
  • General polish — loading states, error messages, empty states
  • Beta test with 10–20 real NYC handball players
  • Fix everything they break
🚀 After Phase 3: Expand courts to other cities. Add direct messaging between players. Add sport selection (basketball, tennis, etc.) to expand beyond handball. The foundation handles all of it without a rebuild.

08 — The Bigger Picture

How It Connects to Your Life OS

The handball app is the foundation. Here's exactly what you build once and reuse in your personal Life OS — so when you're ready to build it, most of the hard work is already done.

Auth System

Built in Phase 1

User accounts, login, sessions, secure data. Your Life OS is just a private account — same system, private screens only you can see.

Database

Built in Phase 1

Supabase Postgres. Add workouts, sleep logs, journal entries, and energy scores as new tables. Zero rebuild — just new data.

Scheduling Logic

Built in Phase 1–2

Date/time logic built for games and tournaments. Reused for training schedule, tutoring blocks, and coding sessions in Life OS.

UI Components

Built across all phases

Cards, charts, stat displays, calendars, profile screens. Life OS reuses the same components — just showing your HRV instead of a score.

🔗 The sequence: Handball app (weeks 1–8) → Math homework tool (weeks 4–6 in parallel) → Life OS (months 3–4, built on top of everything). By the time you build the Life OS, 60–70% of the infrastructure already exists.
